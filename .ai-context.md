# 开发 feature API 接口

> 完整的服务端 API 开发流程，包括路由定义、服务实现、类型定义和测试

## 📋 开发步骤

### 1. 定义共享类型 (Schema)

在 shared/modules 中定义请求和响应的 Schema

**相关文件**:

- `src/shared/modules/feature/schemas.ts`

**检查清单**:

- [ ] 创建请求 Schema (CreateInput, UpdateInput)
- [ ] 创建响应 Schema (ItemSchema, ListSchema)
- [ ] 导出到 src/shared/modules/index.ts
- [ ] 确保使用 Zod 定义类型

**代码示例**:

```typescript
// src/shared/modules/feature/schemas.ts
import { z } from '@hono/zod-openapi'

export const featureSchema = z.object({
  id: z.number(),
  title: z.string(),
  createdAt: z.string(),
})

export const createFeatureSchema = z.object({
  title: z.string().min(1),
})

export type Feature = z.infer<typeof featureSchema>
export type CreateFeature = z.infer<typeof createFeatureSchema>
```

### 2. 创建数据库表（如需要）

在 server/db/schema 中定义数据库表结构

**相关文件**:

- `src/server/db/schema/feature.ts`

**检查清单**:

- [ ] 定义表结构 (pgTable)
- [ ] 定义关系 (relations)
- [ ] 导出到 src/server/db/schema/index.ts
- [ ] 运行 db:generate 生成迁移

**代码示例**:

```typescript
// src/server/db/schema/feature.ts
import { pgTable, serial, varchar, timestamp } from 'drizzle-orm/pg-core'

export const feature = pgTable('feature', {
  id: serial('id').primaryKey(),
  title: varchar('title', { length: 255 }).notNull(),
```

### 3. 实现服务层

在 module 中实现业务逻辑

**相关文件**:

- `src/server/module-feature/services/feature-service.ts`

**检查清单**:

- [ ] 实现 CRUD 操作
- [ ] 处理业务逻辑
- [ ] 使用类型安全的数据库查询
- [ ] 不要在 service 中定义工具函数

**代码示例**:

```typescript
// src/server/module-feature/services/feature-service.ts
import { getDb } from '@server/db'
import { feature } from '@server/db/schema'
import type { CreateFeature } from '@shared/modules/feature/schemas'

export async function listFeatures() {
  const db = await getDb()
  return db.select().from(feature)
}

export async function createFeature(input: CreateFeature) {
  const db = await getDb()
  const result = await db.insert(feature).values(input).returning()
  return result[0]
}
```

### 4. 定义路由

使用 Hono OpenAPI 定义类型安全的路由

**相关文件**:

- `src/server/module-feature/routes/feature-routes.ts`

**检查清单**:

- [ ] 使用 createRoute 定义路由
- [ ] 使用链式语法 .openapi()
- [ ] 调用 service 层方法
- [ ] 使用 successResponse/errorResponse

**代码示例**:

```typescript
// src/server/module-feature/routes/feature-routes.ts
import { OpenAPIHono, createRoute } from '@hono/zod-openapi'
import { featureSchema, createFeatureSchema } from '@shared/modules/feature/schemas'
import { successResponse, errorResponse } from '@server/utils/route-helpers'
import { listFeatures, createFeature } from '../services/feature-service'

const listRoute = createRoute({
  method: 'get',
  path: '/features',
  responses: {
    200: successResponse(z.array(featureSchema), '获取列表'),
  },
})

const createRoute = createRoute({
  method: 'post',
  path: '/features',
  request: {
    body: {
      content: {
        'application/json': { schema: createFeatureSchema },
      },
    },
  },
  responses: {
    201: successResponse(featureSchema, '创建成功'),
    400: errorResponse('参数错误'),
  },
})

export const apiRoutes = new OpenAPIHono()
  .openapi(listRoute, async c => {
    const items = await listFeatures()
    return c.json({ success: true, data: items })
  })
  .openapi(createRoute, async c => {
    const data = c.req.valid('json')
    const item = await createFeature(data)
    return c.json({ success: true, data: item }, 201)
  })
```

## ⚠️ 常见错误

❌ 在路由文件中应用中间件（应在 app.ts 中）
❌ 在 service 中定义工具函数（应放在 utils/）
❌ 使用 any 类型（应使用 Zod Schema）
❌ 直接在路由中实现业务逻辑（应调用 service）
❌ 忘记导出 Schema 到 shared/modules/index.ts

## 开发规范

### 关键规则

1. **中间件位置**: 中间件必须在 `src/server/middleware/` 目录
2. **路由位置**: 路由必须在 `src/server/module-*/routes/` 目录
3. **中间件应用**: 中间件只能在 `app.ts` 中应用，不能在路由文件中应用
4. **类型安全**: 使用 Zod Schema 定义类型，禁止使用 `any`
5. **分层架构**: 路由调用服务，服务处理业务逻辑
6. **工具函数**: 工具函数放在 `utils/` 目录，不要放在 service 中

### API 开发流程

```
1. 定义 Schema (src/shared/modules/{feature}/schemas.ts)
2. 创建数据库表 (src/server/db/schema/{feature}.ts)
3. 实现服务层 (src/server/module-{feature}/services/{feature}-service.ts)
4. 定义路由 (src/server/module-{feature}/routes/{feature}-routes.ts)
5. 注册路由 (src/server/app.ts)
6. 编写测试 (src/server/module-{feature}/__tests__/)
```

### 前端开发流程

```
1. 创建 API 客户端 (src/client/services/apiClient.ts)
2. 创建状态管理 (src/client/stores/{feature}Store.ts)
3. 创建组件 (src/client/components/{feature}/)
4. 创建页面 (src/client/pages/{feature}Page.tsx)
```
