# 智能开发上下文工具使用指南

## 🎯 功能概述

这套工具能够帮助你快速理解项目结构、推导开发链路、生成开发指引，特别适合在新的 AI 会话中快速建立项目认知。

## 📦 包含工具

### 1. 项目上下文分析器 (`dev:context`)

**功能**：扫描项目结构，生成项目上下文文件

**使用方法**：

```bash
# 生成完整的项目上下文
npm run dev:context

# 分析特定功能的开发路径
npm run dev:context "我要开发用户管理接口"
```

**输出文件**：

- `.dev-context.md` - 完整的项目上下文（包含模块、中间件、数据库、API 依赖关系）
- `.dev-context-compact.md` - 紧凑版本（适合复制到 AI 会话）

### 2. 开发指引生成器 (`dev:guide`)

**功能**：根据任务描述生成详细的开发步骤和代码示例

**使用方法**：

```bash
# 生成 API 开发指引
npm run dev:guide "我要开发一个订单管理接口"

# 生成前端开发指引
npm run dev:guide "我要开发一个用户列表页面"

# 生成数据库开发指引
npm run dev:guide "我要创建一个商品表"

# 生成 WebSocket 开发指引
npm run dev:guide "我要开发一个实时聊天功能"

# 生成 SSE 开发指引
npm run dev:guide "我要开发一个消息推送功能"
```

**输出文件**：

- `.dev-guide.md` - 详细的开发指引（包含步骤、检查清单、代码示例）

## 🚀 快速开始

### 场景 1：开始新的 AI 会话

当你开始一个新的 AI 会话时，可以快速让 AI 了解项目：

```bash
# 1. 生成项目上下文
npm run dev:context

# 2. 将紧凑上下文复制到 AI 会话
cat .dev-context-compact.md
# 然后粘贴到 AI 会话中
```

**AI 会话示例**：

```
我正在开发一个 React + Hono 全栈项目。以下是项目上下文：

[粘贴 .dev-context-compact.md 内容]

现在我要开发一个订单管理接口，请帮我实现。
```

### 场景 2：开发新功能

当你需要开发新功能时：

```bash
# 1. 生成开发指引
npm run dev:guide "我要开发一个订单管理接口"

# 2. 查看指引
cat .dev-guide.md

# 3. 按照步骤逐一完成开发
```

### 场景 3：分析现有功能

当你需要了解某个功能的实现细节：

```bash
# 分析特定功能的开发路径
npm run dev:context "订单管理"

# 查看相关的文件和依赖关系
cat .dev-context.md
```

## 📋 生成的上下文包含什么？

### 项目模块

- 模块名称和路径
- 路由文件数量
- 服务文件数量
- 测试文件数量
- 数据库表定义

### API 依赖关系

- API 端点（方法 + 路径）
- 对应的路由文件
- 对应的服务文件
- 使用的 Schema
- 使用的中间件
- 访问的数据库表

### 中间件

- 所有中间件文件列表
- 中间件功能说明

### 数据库

- 表定义文件
- 迁移文件

### 开发规则

- 相关的开发规范文档

## 🎨 开发指引包含什么？

### 开发步骤

每个步骤包含：

1. **标题和描述** - 清晰说明要做什么
2. **相关文件** - 需要创建或修改的文件
3. **检查清单** - 完成步骤需要做的事情
4. **代码示例** - 可直接使用的代码模板

### 常见错误

列出开发过程中容易犯的错误，帮助你避免踩坑

### 测试指南

说明如何测试开发的功能

### 相关文档

指向相关的开发规范文档

## 🔧 工具原理

### 依赖图构建

工具会扫描项目文件，构建依赖关系图：

1. 扫描 `src/server/module-*` 目录
2. 分析路由文件中的 API 定义
3. 分析服务文件中的数据库操作
4. 分析 Schema 文件中的类型定义
5. 构建完整的调用链

### 智能推导

根据任务描述，工具会：

1. 识别任务类型（API/前端/数据库/WebSocket/SSE）
2. 查找相关的现有模块
3. 推导需要的开发步骤
4. 生成对应的代码示例

### 规则匹配

工具会读取 `.claude/rules` 中的开发规范：

1. 匹配相关的规范文档
2. 提取关键的开发要求
3. 生成符合规范的代码示例

## 🤝 与 AI 协作

### 最佳实践

1. **提供上下文**：在 AI 会话开始时，先提供项目上下文
2. **明确需求**：清晰描述你要开发的功能
3. **参考指引**：让 AI 参考生成的开发指引
4. **逐步实现**：按照步骤逐一实现，不要跳步

```

- 创建订单

[AI 会参考 .dev-guide.md 中的步骤和代码示例]

我将按照以下步骤实现：
1. 定义 Schema
2. 创建数据库表
3. 实现服务层
4. 定义路由
...
```
